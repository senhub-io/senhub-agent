# Release 0.3.0-beta

## Overview

0.3.0 is the largest probe expansion in the agent's history and the first
release built on the **float64 value bus**. It adds ~44 new probes — databases,
web/proxy/app servers, message brokers, container/orchestration, hardware and
industrial targets — all shipping in the **free tier**, and wires every one of
them into the **Toise entity model** (stable identity + `monitors` topology).
It lands the post-audit remediation of the 0.3.0 wave and introduces two
regression guards that keep the OTel mapping and the entity wiring honest.

The agent stays a universal edge collector: free basic collection at
competitor parity (Telegraf / Prometheus exporters / otelcol-contrib), feeding
VictoriaMetrics / VictoriaLogs / Toise / Grafana.

## New

### Probes (free tier)
- **Databases**: MySQL, PostgreSQL, SQL Server, Oracle, MongoDB, Redis,
  Cassandra, CouchDB, InfluxDB, ClickHouse, Memcached, Elasticsearch,
  OpenSearch, Solr.
- **Web / proxy / app servers**: Nginx, Apache, HAProxy, Envoy, Varnish,
  PHP-FPM, Tomcat, WildFly.
- **Message brokers**: Kafka (incl. partition ISR), RabbitMQ, NATS, Pulsar,
  ActiveMQ.
- **Coordination / CI / storage**: Consul, ZooKeeper, Jenkins, Ceph.
- **Container / orchestration**: Docker, Kubernetes.
- **Hardware / host**: S.M.A.R.T. disks, IPMI sensors, NVIDIA GPUs, chrony,
  process table, systemd units, Windows services.
- **Industrial / virtualization / network**: Modbus TCP, Proxmox VE, Hyper-V,
  Ubiquiti UniFi.

### Entity model (Toise topology)
- Every monitored target is emitted as a stable entity — `service.instance`
  for application services, `db` for datastores (`db.instance.id` from a stable
  source identifier such as `server_uuid` / `system_identifier` / cluster UUID,
  never a network-derived `host:port`) — plus a `monitors` edge from the agent's
  own `service.instance`.
- Hypervisor-discovered VMs are emitted as `compute.vm` (or a `host` when the
  guest machine-id is available), with a `runs_on` edge.

## Changed

- **Value bus is float64.** `DataPoint.Value` migrated from float32 to float64
  across the whole probe fleet, removing the 24-bit-mantissa precision loss that
  stair-stepped large counters above 2^24 (SNMP / IPMI / Proxmox / 32-bit Modbus
  registers).
- **Free tier grows to 47 probes.** The 0.3.0 wave ships free; Redis, Docker and
  `wifi_signal_strength` moved from the paid catalogue to free. The paid line is
  now only deep vendor infra without a free exporter, the bespoke event
  collector and the active synthetic checks.
- **OTel naming** normalised on several metrics — unit suffixes dropped from
  metric names, `tx/rx` byte metrics collapsed onto a single name with a
  `direction` attribute (see the before/after below).

| Before | After |
|---|---|
| `apache.traffic.bytes` | `apache.traffic` |
| `clickhouse.*.bytes` | `clickhouse.*.data` |
| `unifi.network.tx_bytes` / `rx_bytes` | `unifi.network.io` + `direction` |
| `memcached.network.bytes.{sent,received}` | `memcached.network` + `direction` |

## Fixed

- **Data loss on PRTG/Nagios pull sinks**: multi-instance probes missing a
  `DiscriminantTagsRegistry` entry collapsed their per-instance series onto one
  cache slot — fixed for the whole wave (#459).
- **Security**: MongoDB no longer leaks the connection URI in the `instance`
  tag/logs (#460); UniFi verifies TLS by default (#461); IPMI passes the BMC
  password via `IPMITOOL_PASSWORD` env, never argv (#462).
- **OTel correctness**: HAProxy cumulative metrics are counters not the invalid
  `sum` type (#464); MSSQL buffer-cache hit ratio is a 0-1 ratio (#466).
- **exec timeouts**: smart and ipmi shell-outs are bounded so a hung disk/BMC
  can't stall the poll loop (#463).
- **Metric parity**: PostgreSQL backends collapse + dead-row cleanup (#467);
  Kafka partition ISR is computed (#468); Kubernetes and Proxmox emit
  `senhub.<probe>.up` so an unreachable API is visible (#469).

## Internal

- Two regression guards: OTel `type`/`unit`/naming + discriminant-tags presence
  (#475), and a hardened `EntitySource` invariant — a remote-target probe
  returning the NoOp source now fails CI (#471).
- Test hygiene: systemd helpers made cross-platform testable, kubernetes build
  helpers covered (#473 / #474).
- semconv sections renumbered after the wave merge (#477).

## Known follow-ups

- **Toise coordination** (#479): the `compute.vm` / `container` entities are
  dropped by Toise until those types are registered in its deployment; the
  hyperv/docker metrics and all other entities are unaffected in the meantime.
- **OTel naming gaps** (#480): docker / memcached / nats still carry a unit
  suffix on a metric name; baselined in the guard.
- **Discriminant gap** (#481): `wifi_signal_strength` has multi_instance_labels
  but no DiscriminantTagsRegistry entry.
- **Entity invariant coverage** (#482): per-probe config fixtures so the
  invariant reaches config-requiring probes; a kubernetes cluster
  `service.instance`.
