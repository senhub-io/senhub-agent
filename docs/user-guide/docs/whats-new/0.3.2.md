# SenHub Agent 0.3.2

:material-calendar: 2026-06-17 · probe catalog explosion — ~40 new free-tier probes and the first entity/topology model

The largest release so far. The agent grows from a handful of collectors
into a broad universal probe catalog: around forty new **free-tier**
probes cover relational and NoSQL databases, search engines, message
brokers, web and application servers, containers and virtualization,
hardware sensors and OS internals. Alongside the catalog lands the first
**entity model**: every probe now emits typed nodes and relationships
into the Toise inventory — services, database instances and VMs anchored
to their host, with the first topology edges derived from live host state
(outbound dependencies from the socket table, interface addresses). The
whole probe fleet also moves to a **float64 value bus**, and the licence
format is now JWT-only.

!!! warning "Action may be required"
    0.3.0 and later accept only signed **JWT licences**; the legacy
    compact `SH-XXXX` key format was removed (its signing relied on a
    hardcoded HMAC secret that was cryptographically broken). Only
    **paid** deployments are affected: a paid agent still carrying a
    compact key fails licence validation on upgrade and drops to the
    free tier — every paid probe is disabled. Free-tier installs (no
    `license:` configured) are unaffected. Migrate a JWT licence into
    `agent.license` before upgrading a paid host; the agent now logs at
    `ERROR` and emits `senhub.agent.license.invalid{reason}` so you can
    alert on any host left on a compact key. (#486)

## New

<ul class="rn">
<li><span class="tag t-new">New</span> <span class="tag t-area">databases</span> Relational database probes (free tier): MySQL / MariaDB, PostgreSQL, Microsoft SQL Server and Oracle.</li>
<li><span class="tag t-new">New</span> <span class="tag t-area">databases</span> NoSQL, key-value and columnar store probes (free tier): Redis / Valkey (#393), MongoDB, Memcached, Cassandra, CouchDB, InfluxDB and ClickHouse.</li>
<li><span class="tag t-new">New</span> <span class="tag t-area">search</span> Search-engine probes (free tier): Elasticsearch, OpenSearch and Apache Solr.</li>
<li><span class="tag t-new">New</span> <span class="tag t-area">messaging</span> Messaging and streaming probes (free tier): Apache Kafka, RabbitMQ, Apache ActiveMQ, NATS, Apache Pulsar and Apache ZooKeeper.</li>
<li><span class="tag t-new">New</span> <span class="tag t-area">web</span> Web / application server and proxy probes (free tier): Nginx, Apache, Apache Tomcat (via Jolokia), WildFly / JBoss, HAProxy, PHP-FPM, Varnish, Envoy and Consul.</li>
<li><span class="tag t-new">New</span> <span class="tag t-area">virtualization</span> Container and virtualization probes (free tier): Docker, Kubernetes, Proxmox VE and Hyper-V.</li>
<li><span class="tag t-new">New</span> <span class="tag t-area">hardware</span> Hardware and sensor probes (free tier): S.M.A.R.T. disk health, IPMI / BMC sensors, NVIDIA GPU, Modbus TCP and Chrony (NTP).</li>
<li><span class="tag t-new">New</span> <span class="tag t-area">os</span> Operating-system probes (free tier): Windows services via the SCM, systemd units and a process monitor.</li>
<li><span class="tag t-new">New</span> <span class="tag t-area">infra</span> Additional infrastructure probes (free tier): Jenkins, Ceph and Ubiquiti UniFi Controller.</li>
<li><span class="tag t-new">New</span> <span class="tag t-area">entity</span> Every probe now carries a mandatory <code>EntitySource</code> as a structural invariant, feeding typed nodes into the Toise topology inventory instead of emitting bare metrics. (#471)</li>
<li><span class="tag t-new">New</span> <span class="tag t-area">entity</span> Monitored services and database instances are emitted as <code>service.instance</code> / <code>db.instance</code> entities with a stable identity and a <code>monitors</code> edge from the agent, anchored to their host via <code>runs_on</code>. (#470, #433)</li>
<li><span class="tag t-new">New</span> <span class="tag t-area">entity</span> Hypervisors emit their guests: Hyper-V and Proxmox surface each VM as a <code>compute.vm</code> entity keyed on the host machine identity, with <code>runs_on</code> and <code>monitors</code> edges. (#470, #456, #433)</li>
<li><span class="tag t-new">New</span> <span class="tag t-area">topology</span> Outbound dependency edges are derived from the kernel socket table: an <code>ESTABLISHED</code> connection owned by a local process becomes a durable <code>depends_on</code> edge to an observable <code>network.endpoint</code>, debounced across scrapes so the graph carries topology, not ephemeral flow. (#457)</li>
<li><span class="tag t-new">New</span> <span class="tag t-area">topology</span> The host's own interface addresses are modelled: each non-loopback interface becomes a <code>network.interface</code> and each unicast IP a <code>network.address</code>, so a connection peer resolves back to the remote host's listener. (#458)</li>
<li><span class="tag t-new">New</span> <span class="tag t-area">entity</span> The process monitor emits a <code>process</code> entity per monitored process (keyed on <code>process.pid</code> + creation time) with a <code>runs_on</code> edge, in inventory mode only.</li>
<li><span class="tag t-new">New</span> <span class="tag t-area">checks</span> <code>http_check</code> gains a TLS certificate validity metric with issuer and subject attributes.</li>
<li><span class="tag t-new">New</span> <span class="tag t-area">CLI</span> New <code>refresh-unit</code> subcommand to update an installed systemd unit in place.</li>
</ul>

## Improved

<ul class="rn">
<li><span class="tag t-improved">Improved</span> <span class="tag t-area">docker</span> The Docker probe reaches full OTel receiver parity — per-core CPU, throttling, deep memory and blkio breakdowns.</li>
<li><span class="tag t-improved">Improved</span> <span class="tag t-area">redis</span> The Redis probe reaches full OTel receiver parity plus cluster, sentinel, tracking and community metrics.</li>
<li><span class="tag t-improved">Improved</span> <span class="tag t-area">metrics</span> The whole probe fleet moved to a <strong>float64 value bus</strong>: datapoint values are no longer clamped to float32 precision. (#271, #407)</li>
</ul>

## Fixed

<ul class="rn">
<li><span class="tag t-fixed">Fixed</span> <span class="tag t-area">probes</span> Probes that split a metric across tag values now register their discriminant tags, so pull sinks (Prometheus, web UI) no longer collapse distinct series into one. (#459)</li>
<li><span class="tag t-security">Security</span> <span class="tag t-area">databases</span> Credential-bearing probes verify TLS by default and stop smuggling secrets: SQL Server uses a URL DSN with TLS on, PostgreSQL uses a URL DSN with opportunistic TLS, UniFi verifies TLS on login (#461), and MongoDB derives a credential-free instance string and never emits or logs the raw URI (#460).</li>
<li><span class="tag t-security">Security</span> <span class="tag t-area">hardware</span> The IPMI probe passes its password via the <code>IPMITOOL_PASSWORD</code> environment variable instead of the command line. (#462)</li>
<li><span class="tag t-fixed">Fixed</span> <span class="tag t-area">hardware</span> The S.M.A.R.T. and IPMI probes bound their external tool executions with a context timeout (and <code>WaitDelay</code> for a forking child), so a stuck <code>smartctl</code> / <code>ipmitool</code> can't stall the poll loop. (#463)</li>
<li><span class="tag t-fixed">Fixed</span> <span class="tag t-area">virtualization</span> Kubernetes and Proxmox emit <code>senhub.&lt;probe&gt;.up</code> so an unreachable API surfaces as down instead of silently disappearing. (#469)</li>
<li><span class="tag t-fixed">Fixed</span> <span class="tag t-area">databases</span> PostgreSQL and MySQL tolerate a target that is down at startup instead of failing to register. (#485)</li>
<li><span class="tag t-fixed">Fixed</span> <span class="tag t-area">databases</span> PostgreSQL collapses its backend counts to a bare metric and drops never-emitted YAML rows (#467); Kafka computes and emits partition ISR (replicas in sync) (#468); SQL Server's buffer-cache hit ratio is reported as a 0–1 ratio (<code>otel.unit 1</code>). (#466)</li>
<li><span class="tag t-fixed">Fixed</span> <span class="tag t-area">metrics</span> Several probe metrics dropped a redundant unit suffix from the metric name (Apache, ClickHouse, Memcached, UniFi), and cumulative HAProxy metrics are now typed as counters rather than an invalid sum. (#464, #465)</li>
<li><span class="tag t-fixed">Fixed</span> <span class="tag t-area">licensing</span> A configured-but-rejected licence now fails loud — logged at <code>ERROR</code> with a <code>senhub.agent.license.invalid{reason}</code> self-metric — instead of silently degrading to the free tier. (#486)</li>
<li><span class="tag t-fixed">Fixed</span> <span class="tag t-area">Nagios</span> The Nagios output reports <code>CRITICAL</code> when a probe's <code>up</code> metric is 0.</li>
<li><span class="tag t-fixed">Fixed</span> <span class="tag t-area">Prometheus</span> The <code>prometheus_scrape</code> probe no longer double-prefixes its own self-scrape, closing a naming loop.</li>
<li><span class="tag t-fixed">Fixed</span> <span class="tag t-area">OTLP</span> <code>service.instance.id</code> uses the full agent key; a replaced strategy is shut down before its recreation on config reload.</li>
<li><span class="tag t-fixed">Fixed</span> <span class="tag t-area">topology</span> Host-dependency dependents are anchored to their host with a <code>runs_on</code> edge. (#496)</li>
</ul>
