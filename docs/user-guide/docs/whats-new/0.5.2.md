# SenHub Agent 0.5.2

:material-calendar: 2026-07-10 · PowerStore depth, Veeam plugin jobs, pull-view ratio fix

A follow-up to 0.5.1 that deepens the Dell PowerStore probe, fixes how
percentage ratios render in the PRTG/Nagios pull views across many probes, and
corrects Veeam job typing for plugin-managed backups.

<div class="rn-filter"></div>


## Improved

<ul class="rn">
<li><span class="tag t-improved">Improved</span> <span class="tag t-area">PowerStore</span> The PowerStore probe now emits <strong>per-resource series</strong> — one per volume, appliance, node, drive and replication session — each carrying a resource attribute that also acts as a filter in the Web UI Sensor Builder. Cluster-level aggregates are complemented by per-appliance and per-node capacity and performance.</li>
<li><span class="tag t-improved">Improved</span> <span class="tag t-area">PowerStore</span> Latency is now reported in <strong>milliseconds</strong> in the pull views (exported as seconds over OTLP), and appliance health is derived from its component faults rather than a lifecycle enum.</li>
</ul>


## Fixed

<ul class="rn">
<li><span class="tag t-fixed">Fixed</span> <span class="tag t-area">PRTG</span> <span class="tag t-area">Nagios</span> Metrics exported as <code>0..1</code> ratios (OTel unit <code>1</code>) are now shown correctly as percentages in the PRTG/Nagios pull views. This corrected a double-scaling across many probes — process, SMART, MySQL, PostgreSQL, Oracle, Redis, Elasticsearch, OpenSearch, Proxmox, NVIDIA, Hyper-V and ActiveMQ.</li>
<li><span class="tag t-fixed">Fixed</span> <span class="tag t-area">Veeam</span> Plugin-managed backups (Veeam Backup for <strong>Nutanix AHV</strong> and other external modules) are now labelled <code>CustomPlatform</code> — Veeam's own term — instead of the alarming <code>Unknown</code>, so they group and filter cleanly.</li>
<li><span class="tag t-fixed">Fixed</span> <span class="tag t-area">PRTG</span> Probe names in the pull views are matched case-insensitively and rendered verbatim, so a lower-cased probe name in the config no longer loses its sensor filters or shows a mismatched capitalisation. (#625, #626)</li>
</ul>


## Security

<ul class="rn">
<li><span class="tag t-security">Security</span> Built with Go 1.26.5, picking up the <code>crypto/tls</code> fix (GO-2026-5856).</li>
</ul>
