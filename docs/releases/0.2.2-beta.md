# Release 0.2.2-beta

**Date:** 2026-06-11
**Type:** Feature release — active checks, custom checks, Prometheus scraping, PRTG migration path. No breaking changes.

## Overview

0.2.2 completes the PRTG-replacement funnel opened by 0.2.1. Three axes:

1. **Active checks (free).** Four multi-target check probes on a shared
   bounded fan-out chassis: `icmp_check` (ping, packet loss, RTT),
   `http_check` (status, latency phases, content match, TLS certificate
   expiry in days), `tcp_dial` (raw connect latency), `dns_latency`
   (per-name, per-resolver). A failing target is a measurement
   (`up = 0`), never a probe failure.
2. **Universal collection grows (free).** `prometheus_scrape` pulls
   /metrics endpoints into the pipeline (typed pass-through,
   counter/gauge preserved); `exec` runs operator-supplied checks with
   the Nagios plugin contract (existing `check_*` plugins work
   unchanged) or a JSON contract; `syslog` moves from Pro to free.
3. **Migration path.** A "Migrating from PRTG" guide (sensor-to-probe
   mapping, progressive vs direct path, honest gaps), documentation
   pages for all five 0.2.1 collectors, and ready-to-import alerting
   starter packs (vmalert + Grafana) under `packs/alerts/`.

Free tier: 11 -> 17 probes.

## New

- `icmp_check` — multi-target ping: reachability, packet loss, RTT
  min/avg/max/stddev (#299, PR #339)
- `http_check` — HTTP(S) checks: status validation, latency phases
  (DNS / connect / TLS / TTFB), response size, content matching, TLS
  expiry as a first-class metric (#300, PR #340)
- `tcp_dial` — TCP connect latency to host:port (#159, PR #342)
- `dns_latency` — resolution latency per name x resolver pair (#158,
  PR #342)
- `prometheus_scrape` — scrape Prometheus expositions into the
  pipeline; `metric_match` cardinality filter, bearer token, 32 MiB
  cap; histogram/summary series dropped and counted (#304, PR #349)
- `exec` — custom checks: Nagios plugin contract (exit code +
  perfdata, UOM normalization) and JSON contract; absolute-path +
  world-writable refusal, no shell, hard timeout with process-tree
  kill, overlap guard (#305, PR #350)
- Alerting starter packs under `packs/alerts/`: vmalert rules for
  host / active checks (TLS expiry at 30 and 7 days) / SNMP / agent
  self-health, plus a Grafana provisioning example; validated with
  `vmalert -dryRun` (#301, PR #347)
- "Migrating from PRTG" user-guide section + doc pages for snmp_poll,
  snmp_trap, filetail, windows_eventlog, otlp_receiver (#302, PR #346)

## Changed

- `syslog` moves from Pro to the **free tier** — completes the
  universal log-collection set alongside `filetail` and
  `windows_eventlog` (#298, PR #351)
- Prometheus exposition: `senhub_httpcheck_status_code` and
  `senhub_snmp_interface_{admin,oper}_status` no longer carry a
  misleading `_ratio` suffix (annotation units `{code}` / `{status}`).
  Neither name had shipped in a stable release, so no rename table is
  needed (PR #344)

## Fixed
