# Release 0.2.1

**Date:** TBD (beta re-cut 2026-06-10, tag `0.2.1-beta` @ enterprise `743faed`)
**Type:** Feature release ‚Äî entity rail, network topology, free-tier expansion. One breaking change (license format).

## Overview

0.2.1 turns the agent into a topology-aware edge collector. Three axes:

1. **Entity rail (OTel entity-events, merged spec).** The agent emits
   `entity.state` / `entity.delete` events over the OTLP logs transport:
   host, service.instance, service.listener, and the full SNMP/host network
   topology (device, interface, route, address) with embedded
   `entity.relationships` and removal-by-absence semantics.
2. **Correlation backbone.** The same identity keys ride all three signals:
   `host.id` + `host.*`/`os.*` resource attributes on metrics and logs
   (auto-detected), `network.device.id` + `interface.name` tags on SNMP
   metrics. Any entity in the graph joins its telemetry by exact key.
3. **Free-tier expansion (open-core).** Five new probes ship free:
   `snmp_poll`, `snmp_trap`, `windows_eventlog`, `filetail`, `otlp_receiver`.

Runtime-validated end-to-end before the cut: two production agents (Linux +
Windows Server 2022) feeding a Toise instance through a public OTLP ingress
(nginx ‚Üí otelcol), 87 entities / 83 relations live, identities stable across
cycles.

## Breaking changes

### Compact license format removed ‚Äî JWT only

The 40-character compact license format is gone (it relied on a hardcoded
HMAC secret, unacceptable for an open-source code base). The agent now
accepts only RS256-signed JWT licenses.

**Operators with a compact license MUST replace it before upgrading**, or the
agent silently degrades to the free tier at startup (paid probes are skipped
with a `üö´ Probe not authorized` warning ‚Äî the service itself stays up):

```yaml
# agent.yaml
agent:
  license: "${file:/etc/senhub-agent/license.jwt}"
```

Free-tier-only deployments need no license: remove the `license:` line to
avoid the startup warning.

### Reminder: service command line requires the `run` subcommand

Services installed before 0.2.0 may still have a binPath / ExecStart without
the `run` subcommand. The binary exits immediately when launched without it,
which on Windows surfaces as an SCM timeout ("service did not connect").
Verify before an in-place binary swap (#309 tracks automatic migration):

```
sc.exe config senhub-agent binPath= "C:\SenHub\senhub-agent.exe run --config-path C:\SenHub\agent.yaml"
```

## New probes (all free tier)

| Probe | What it does |
|---|---|
| `snmp_poll` | SNMP v2c/v3 polling ‚Äî MIB-II / IF-MIB metrics + network topology entities (LLDP, routes, addresses) |
| `snmp_trap` | SNMP v2c/v3 trap receiver ‚Üí OTLP logs, operator-supplied local MIBs |
| `windows_eventlog` | Windows Event Log channels ‚Üí OTLP logs (bookmarks, levels filter) |
| `filetail` | Flat-file log tailing ‚Üí OTLP logs (fingerprinting, rotation-safe) |
| `otlp_receiver` | Embedded OTLP receiver ‚Äî the agent as edge collector (gRPC/HTTP in ‚Üí all sinks) |

## Entity rail

- Events follow the merged OTel entity-events spec: `EventName`
  (`entity.state` / `entity.delete`), bare keys (`entity.type`, `entity.id`,
  `entity.description`, `entity.report.interval`).
- Relationships are embedded in the state event (`entity.relationships`
  array, bare descriptors `{relationship.type, entity.type, entity.id}`);
  a relation the source stops listing is removed ‚Äî no explicit edge deletes.
- Entity types emitted: `host`, `service.instance`, `service.listener`
  (listening TCP sockets with process attribution), `network.device`,
  `network.interface`, `network.route`, `network.address`.
- Relation types: `runs_on`, `has_interface`, `has_route`, `bound_to`,
  `next_hop_via`, `connected_to` (LLDP port‚Üîport adjacency).
  `adjacent_to`/`routes_via`/`forwards_to` are no longer emitted (superseded;
  the BRIDGE-MIB FDB walk was removed).
- `network.device.id` precedence: `serial:` > `engine:` > `mac:` > `name:` >
  `mgmt:`. Readable attributes on devices: `sys.name`, `mgmt.ip`,
  `device.role`, `vendor`.
- Enable with one switch on the OTLP strategy: `signals.entities.enabled:
  true` (events ride the logs transport ‚Äî no separate endpoint).

## Correlation

- The OTLP resource now carries `host.id`, `host.name`, `host.arch`, and
  `os.*` attributes on **all** signals, auto-detected (`/etc/machine-id` on
  Linux, registry MachineGuid on Windows) ‚Äî the same `host.id` that
  identifies the host entity.
- SNMP metrics are tagged `network.device.id` (all points) and
  `interface.name` (per-interface points) ‚Äî joining interface traffic to its
  `network.interface` entity by exact key.

## SNMP discovery (foundation)

The seed-based discovery engine ships in this release (crawl engine +
`discovery:` config block: seeds, single v2c profile, mandatory
`allowed_cidrs`, `max_devices`/`max_hops` bounds). Multi-target probe
integration ‚Äî actually polling what the crawl finds ‚Äî is follow-up work
(#156 Lot C, needs a real device fleet).

## OTLP transport hardening

- **Durable logs queue** ‚Äî log batches survive agent restarts
  (disk-backed, replayed on startup).
- **Endpoint failover** ‚Äî secondary OTLP endpoint with automatic return to
  primary.
- Entities signal with its own interval and buffer.

## Other
