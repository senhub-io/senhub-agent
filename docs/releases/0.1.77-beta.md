# Release 0.1.77-beta

**Date:** 2025-12-10
**Type:** Beta Release
**Branch:** dev

## Overview

This release implements a critical online mode bypass for the license system, allowing online agents (connected to SenHub platform) to operate as Enterprise tier without explicit JWT license tokens. This temporary solution enables seamless deployment while the backend license distribution system is being developed.

## Major Changes

### License System: Online Mode Bypass

**Implementation:**
- Online agents (`server_url` configured) now function as **Enterprise tier** (all probes authorized)
- Offline agents (no `server_url`) require explicit JWT license tokens
- Dual nil check protection ensures security (validator + license state machine)

**Technical Details:**
- Modified `IsProbeAuthorized()` in both `JWTValidator` and `MockValidator`
- `license == nil` → returns `true` (Enterprise behavior)
- Backward compatible with existing offline deployments
- Zero downtime upgrade path

**Security:**
- Dual layer protection prevents abuse:
  - No validator → Free tier only (safe mode)
  - Validator + No license → Enterprise (online mode, backend-controlled)
  - Validator + License → Tier from JWT (offline mode)
- Platform authentication protects online mode
- Offline mode still requires cryptographic JWT verification

## Migration Guide

### For Online Agents (SaaS)

**No action required.** Online agents will automatically operate as Enterprise tier:

```yaml
agent:
  authentication_key: "your-uuid"
  server_url: "https://eu-west-1.intake.senhub.io"
  # NO license field needed - online mode = Enterprise
```

### For Offline Agents (Self-Hosted)

**No change.** Continue using JWT license tokens or free tier:

```yaml
agent:
  authentication_key: "your-uuid"
  license: "eyJhbGci..." # JWT token (optional)
```

Without license field: Free tier (cpu, memory, logicaldisk, network)
With license field: Tier from JWT (Free, Pro, or Enterprise)

1. **Backend Integration Pending:** Online agents bypass license checks until backend implements JWT distribution via configuration API

3. **Offline Mode Unchanged:** Offline agents still require manual JWT token management
