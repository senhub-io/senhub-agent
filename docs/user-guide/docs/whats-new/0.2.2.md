# SenHub Agent 0.2.2

:material-calendar: 2026-06-11 · feature release

The PRTG-replacement funnel is complete: free **active checks** (ping,
HTTP with TLS-expiry tracking, TCP, DNS), an **exec probe** that runs
existing Nagios plugins unchanged, **Prometheus endpoint scraping**, a
sensor-to-probe **migration guide** with ready-to-import alerting
packs — and the free tier grows to 17 probes. No breaking changes.

!!! highlight "Highlights"
    - **Six new active and custom checks — all free.** Ping, HTTP(S) with
      TLS-expiry tracking, TCP connect, DNS resolution, Prometheus
      endpoint scraping, and an exec probe that runs your existing Nagios
      plugins unchanged.
    - **A real PRTG migration path.** A sensor-to-probe
      [migration guide](../prtg-migration.md) and ready-to-import
      [alerting starter packs](https://github.com/senhub-io/senhub-agent/tree/dev/packs/alerts)
      (vmalert + Grafana) covering every free probe.
    - **Syslog goes free.** The syslog receiver joins the universal
      collection tier.

## New

<ul class="rn">
<li><span class="tag t-new">New</span> <span class="tag t-area">checks</span> <code>icmp_check</code> probe — multi-target ping: reachability, packet loss, RTT statistics. <em>(Free tier.)</em> (#299)</li>
<li><span class="tag t-new">New</span> <span class="tag t-area">checks</span> <code>http_check</code> probe — HTTP(S) checks: status, latency phases (DNS / connect / TLS / TTFB), content match, and TLS certificate expiry in days as a first-class metric. <em>(Free tier.)</em> (#300)</li>
<li><span class="tag t-new">New</span> <span class="tag t-area">checks</span> <code>tcp_dial</code> probe — raw TCP connect latency to any host:port. <em>(Free tier.)</em> (#159)</li>
<li><span class="tag t-new">New</span> <span class="tag t-area">checks</span> <code>dns_latency</code> probe — resolution latency per name, per resolver (system or explicit). <em>(Free tier.)</em> (#158)</li>
<li><span class="tag t-new">New</span> <span class="tag t-area">checks</span> <code>exec</code> probe — run any Nagios plugin or JSON-emitting script on interval: hard timeout with process-tree kill, overlap protection, perfdata parsing. <em>(Free tier.)</em> (#305)</li>
<li><span class="tag t-new">New</span> <span class="tag t-area">Prometheus</span> <code>prometheus_scrape</code> probe — scrape /metrics endpoints (node_exporter, appliance exporters) into the agent pipeline, counter/gauge semantics preserved. <em>(Free tier.)</em> (#304)</li>
<li><span class="tag t-new">New</span> <span class="tag t-area">alerting</span> Alerting starter packs — vmalert rule files (host, active checks incl. TLS expiry, SNMP, agent self-health) plus a Grafana provisioning example, under <code>packs/alerts/</code>. (#301)</li>
<li><span class="tag t-new">New</span> <span class="tag t-area">docs</span> "Migrating from PRTG" guide — sensor-to-probe mapping, progressive vs direct path, alerting equivalents, honest gaps — plus documentation pages for all five 0.2.1 collectors. (#302)</li>
</ul>
