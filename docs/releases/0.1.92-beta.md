# Release 0.1.92-beta

**Date:** 2026-05-13
**Type:** Beta Release (Database probes OTel-first refactor)
**Branch merged to dev:** `feat/db-otel-first`

## Overview

This release completes the OTel-first migration for the `mysql` and
`postgresql` probes. All metric names have been replaced with
OTel-canonical names. Downstream PRTG and Nagios output is unchanged
(the `display_name` transformer handles the human-readable labels);
Prometheus and OTLP consumers will see new names from 0.1.92 onwards.

Four PRs merged:

1. **mysql: OTel-first metric names** ŌĆö replace the legacy `db_*`
   flat namespace with `mysql.*` (upstream OTel semconv) and
   `senhub.db.mysql.*` (proprietary extensions).
2. **postgresql: OTel-first metric names** ŌĆö same treatment with
   `postgresql.*` and `senhub.db.postgresql.*` / `senhub.db.*`.
3. **`sanitize.Bytes` ŌĆö lift 2 GB cap** ŌĆö `sanitize.BytesInt32`
   renamed to `sanitize.Bytes`; byte counters flow with float32
   precision (~1 KB error at 1 GB, ~256 MB at 1 PB) instead of
   saturating at 2 GB.
4. **DiscriminantTagsRegistry** ŌĆö mysql + postgresql entries added
   so per-database tag-discriminated metrics produce stable cache
   keys.

### Unit changes (breaking for Prometheus / OTLP consumers)

- `row_lock.time.avg`: was milliseconds, now **seconds** (`s`).
  Prometheus name changes from `ŌĆ”avg_ms` to `ŌĆ”avg_seconds`.
- `stat_statements.exec_time.mean`: was milliseconds, now **seconds**.
  PRTG/Nagios display values are unchanged (the `display_name`
  transformer converts to the human-readable unit before delivery).

### OTel resource attributes

Every MySQL and PostgreSQL datapoint now carries three OTel resource
attributes in addition to the existing probe tags:

| Attribute | Example |
|---|---|
| `db.system.name` | `mysql` or `postgresql` |
| `server.address` | `db.example.com` |
| `server.port` | `3306` / `5432` |

### `sanitize.BytesInt32` renamed to `sanitize.Bytes`

Internal API change. Any custom probe code calling
`sanitize.BytesInt32(v)` must be updated to `sanitize.Bytes(v)`.

## Migration guide

### Prometheus / PromQL

Update alert rules, recording rules, and dashboard queries.
Search-and-replace the old metric names using the tables above.
The Prometheus name mapping follows the standard OTel-to-Prometheus
rules (dots ŌåÆ underscores, `senhub_` prefix, unit suffix).
Examples:

```promql
# MySQL active threads
- senhub_db_connections_active
+ mysql_threads{kind="running"}

# PostgreSQL commits
- senhub_db_transactions_committed_total
+ postgresql_commits_total

# MySQL deadlocks
- senhub_db_locks_deadlocks_total
+ senhub_db_mysql_lock_deadlocks_total

# MySQL IO (old single metric split into attribute)
- senhub_db_io_read_bytes_total
+ senhub_db_mysql_io_bytes_total{io_direction="read"}
```

### OTLP consumers (vmagent, Grafana Cloud, Tempo)

The OTLP push payload changes from `system.db.*` / `db_*` style to the
OTel-canonical names listed above. Update any receiver-side
transformations, dashboards, or pipelines that reference the old names.

### PRTG / Nagios

No action required. The `display_name` transformer maps new OTel
metric names back to the human-readable channel names used in sensor
templates. Existing sensors continue to display correct values.

## Non-breaking additions

- `senhub.db.replication.role` lookup now registered in PRTG
  lookup catalog: `0=STANDALONE`, `1=PRIMARY`, `2=REPLICA`.
- mysql + postgresql entries added to `DiscriminantTagsRegistry`
  for stable cache keys when `expose_per_database: true`.
