# Release 0.1.93-beta

**Date:** 2026-05-17
**Type:** Beta Release (IBM i probe + multi-file config + license hardening)
**Branches merged to dev:** `chore/claude-rules`, `feat/conf-multifile-envsubst`, `fix/orphan-transformers`, `feat/ibmi-probe`, `feat/ibmi-otel-coverage`, `chore/0.1.93-beta-prep`

## Overview

Six PRs merged since 0.1.92-beta, grouped in three themes:

1. **IBM i / Power Systems probe** (PR #105 + #106) — new probe family with 28+ collectors, JT400 JDBC bridge, 94 metrics covering CPU, ASP, jobs, queues, journals, security, hardware, network and self-observability. Full OTel-first mapping under `senhub.ibmi.*` (Lot 5, modelled on veeam/citrix/netscaler).
2. **Multi-file configuration layout** (PR #103) — `agent.yaml` + `probes.d/*.yaml` + `strategies.d/*.yaml` with `${env:VAR}` / `${file:/path}` substitution, plus `agent config show` CLI. Auto-detected, legacy single-file installs continue to work unchanged.
3. **License system hardening + doc sync** (this PR `chore/0.1.93-beta-prep`) — closes a drift gap that motivated the work: linux_logs, mysql, postgresql and ibmi were registered without a compact-license bitmap slot. Adds a structural test that makes the drift fail CI in future.

Two cleanups land alongside:
- PR #102 — `CLAUDE.md` split into path-scoped `.claude/rules/*.md` modules (loaded contextually per file).
- PR #104 — drop three orphan transformer YAMLs (`gateway.yaml`, `webapp.yaml`, `storage.yaml`) that were embedded in every binary but never loaded by the registry.

## New features

### IBM i / Power Systems probe (Pro tier)

- Registry: `"ibmi"` → `ibmi.NewIBMiProbe`.
- Bridge: spawns `java -cp jt400.jar:. Jt400Runner` (default; requires a JRE on the host) or a precompiled `jt400runner` native binary when `bridge.native_runner_path` is set. Subprocess auto-respawn on failure, handshake via JSON-per-line over stdout.
- Collectors: active_job, asp, audit_journal, disk_status, hardware_resource, history_log, http_server, index_advisor, job_queue, journal, jvm, library_list, memory_pool, message_queue, msgw_job, netstat_*, output_queue, ptf_group, query_supervisor, scheduled_job, spooled_file, sys_table_stats, system_status, system_value, user_profile, user_storage, watch_info, plus the bridge self-observability.
- 94 metrics in `definitions/ibmi.yaml`, all OTel-mapped. Naming follows `senhub.ibmi.<family>.<measure>`. 4 event-conduit metrics (msgw_job, message_queue, history_log, audit_journal) carry `otel.skip: true` — same pattern as syslog/event; future OTLP logs export targets V2.
- Probe-side units convert automatically (`%`→`1`, `KB`/`MB`→`By`, `ms`→`s`, `B`→`By`). One non-canonical unit (`ms/s` for `senhub.ibmi.job.cpu.rate`) uses explicit `otel.value_scale: 0.001` to produce a dimensionless ratio.
- `tag_to_attribute` renames probe cache tags to clean OTel attribute keys under `ibmi.*`. `network.transport` reused for the canonical OTel attribute key.
- Full catalog documented in `docs/developer-guide/otel/senhub-semantic-conventions.md` §4.14.

### Multi-file configuration layout

- Operators can now split `agent-config.yaml` into:
  ```
  agent.yaml             # global keys only (agent, cache, auto_update, log)
  probes.d/
    10-system.yaml       # array of probe configs
    20-database.yaml
    90-disabled.yaml.disabled    # ignored
  strategies.d/
    prtg.yaml            # one strategy per file
    otlp.yaml
    senhub.yaml
  ```
- Alphabetic ordering of `.d/` files. Files matching `.*` or `*.disabled` are skipped. Empty `.d/` directories are valid.
- **Legacy installs continue to work**: if `agent.yaml` contains a top-level `probes:` or `storage:` block, the monolithic path is taken and `.d/` directories are ignored with a one-time WARN log.
- Duplicate strategy across multiple files: later file wins, WARN logged.

### Environment + file substitution

- `${env:VAR}` and `${env:VAR:-default}` — POSIX-shell parity for env-resolved values.
- `${file:/path}` and `${file:/path:-default}` — read value from a file path, `TrimSpace`d (so operators can write `echo "secret" > file` without trailing-newline issues).
- `$$` escapes to a literal `$`.
- Substitution runs **after** the merge: an inline reference inside a `probes.d/` fragment resolves against the agent's environment the same way an inline reference in the legacy monolithic file would.
- Mutates string **values only** — keys are never substituted, by design.

### `agent config show` CLI

- `agent config show` prints the merged + substituted configuration (the exact tree the agent boots with).
- `--raw` — preserve `${...}` references unsubstituted, for audit.
- `--redact` — mask values coming from `${file:...}` AND values whose key matches `(?i)(key|token|password|secret)` with `***`. Safe to paste into a support ticket or commit to source control.
- yaml.v3 output with alphabetically sorted map keys → byte-identical output on N runs, diffable.

### IBM i OTel coverage (Lot 5) + linux_logs semconv §4.15

- 94/94 ibmi metrics OTel-mapped on PR #105's heels.
- New §4.14 documents the full ibmi catalog by family with name/unit/type/attributes.
- New §4.15 promotes `linux_logs` from a single-line mention to its own dedicated section: full attribute catalog (host.name, systemd.unit, syslog.appname, process.*, etc.), body/timestamp policy, the RFC 5424 → OTel SeverityNumber + SeverityText table (shared with syslog/event), upstream filtering knobs, and why there is no synthetic DataPoint (logs-only, no metric path).

## Internal — license system hardening

Four probes (`linux_logs`, `mysql`, `postgresql`, `ibmi`) were registered between 0.1.91 and 0.1.93 without anyone updating the compact-license bitmap (`probeBitmap` in `internal/agent/services/license/compact.go`). The compact-license format could therefore not authorize them — only the longer JWT format worked, the inverse of why the compact format exists. The `docs/LICENSE-SYSTEM.md` tier sections had drifted as well, still listing the removed `otel` probe and missing the four newer probes.

Changes:

- **`probeBitmap` extended** with bits 15 (mysql), 16 (postgresql), 17 (linux_logs), 18 (ibmi). 13 bits (19-31) remain reserved for future probes.
- **Two new structural tests** in `internal/agent/probes/registry_invariant_test.go`:
  - `TestEveryRegisteredProbeIsAuthorizable` — every probe in the registry must be in `freeTierProbes` OR `probeBitmap`. CI fails on drift.
  - `TestNoStaleBitmapEntry` — every probe claiming a bitmap slot must still be in the registry. Catches the reverse direction (e.g. a bit pointing at a removed probe).
- **Two new exported helpers** in the license package — `license.IsProbeAuthorizable(name) bool` and `license.CompactBitmapProbeNames() []string`. These answer structural questions ("can any license authorize this probe?") without needing a live JWT.
- **`docs/LICENSE-SYSTEM.md` tier sections refreshed** — `otel` removed, `veeam` / `mysql` / `postgresql` / `ibmi` added to Pro, `linux_logs` moved to Free (matching the code).
- **`.claude/rules/probes.md`** gains an explicit numbered checklist of the four touch-points every new probe must update — registry, license tier (free or bitmap), `LICENSE-SYSTEM.md` doc, YAML transformer with `otel:` blocks. The first two are CI-enforced from this release on.

## Internal — codebase hygiene

- **CLAUDE.md split** (PR #102) — the universal Go/git/build/commit rules moved to user-level `~/.claude/CLAUDE.md`; project-specific rules now live under `.claude/rules/*.md` with `paths:` frontmatter for path-scoped loading. CLAUDE.md root file is now a project index.
- **Three orphan transformer YAMLs dropped** (PR #104) — `gateway.yaml`, `webapp.yaml`, `storage.yaml` (-670 lines). They were embedded in every binary but never loaded by the registry — leftovers from before the `ping_gateway` / `ping_webapp` / `load_webapp` split. Dell PowerVault ME (`storage.yaml`) was a design draft, the probe was never implemented.

## Known follow-ups

1. **License tier decision for ibmi customers**: the probe ships in the binary and the compact bitmap can authorize it now, but the demo JWT in `dist/agent-config.yaml` only authorizes `[citrix, netscaler]`. Customers wanting ibmi need a custom JWT or a compact license that includes `ibmi`. Demo JWT update requires re-signing in Sensor Factory (not regenerable from the agent repo).
2. **Real-IBM-i smoke test** — needs access to a target (Heliaq or other) to validate end-to-end `senhub_ibmi_*` Prometheus emission and OTLP push.
3. **Compact license bitmap width** — at the rate of ~5 new probes per release we have ~3 releases before claiming bit 31. When the time comes, bump the format version byte (`compactVersion`) from `0x01` to `0x02` and grow the bitmap to `uint64`.
4. **Zabbix output** — Phase 0 audit landed; Phase 1 starting next release window. Not advertised in user-facing docs yet.
