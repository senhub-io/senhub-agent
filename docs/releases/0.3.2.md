# Release 0.3.2

**Type:** First stable of the 0.3.x line. Consolidates the 0.3.0 probe
expansion and the 0.3.1 / 0.3.2 connection-topology work into one stable
release. One breaking change for paid deployments (carried from 0.3.0 —
see below).

## Overview

0.3.2 is the first stable release built on the **float64 value bus** and the
**Toise entity model**. It promotes the largest probe expansion in the agent's
history — ~47 free-tier probes spanning databases, web/proxy/app servers,
message brokers, container/orchestration, hardware and industrial targets — and
adds **connection topology**: the agent now models its own network position
(interfaces, addresses) and its **outbound dependencies** ("who talks to whom")
as first-class graph entities, on a hardened entity-emission path.

The agent stays a universal edge collector: free basic collection at competitor
parity (Telegraf / Prometheus exporters / otelcol-contrib), feeding
VictoriaMetrics / VictoriaLogs / Toise / Grafana.

Upgrading from 0.2.x: the full 0.3.0 probe catalogue, entity model, float64 bus
and OTel naming changes are documented in
[`0.3.0-beta.md`](./0.3.0-beta.md) — that note's **Breaking changes**, **New**
and **Changed** sections all apply to this stable. This note details what 0.3.2
adds on top.

## Breaking changes

**Legacy compact licence keys (`SH-XXXX-…`) are no longer accepted (#486).**
Carried from 0.3.0 and still in force. 0.3.x validates only signed JWT licences;
the compact key format was removed because its signing relied on a hardcoded
HMAC secret that was cryptographically broken — keeping it would let anyone mint
a valid key, so its removal is a security fix.

- **Who is affected.** Only **paid** deployments. A paid agent still carrying a
  compact `SH-XXXX` licence will, on upgrade, fail validation and **drop to the
  free tier — every paid probe is disabled**. Free-tier agents (no `license:`)
  are unaffected.
- **It is not silent.** The agent logs at `ERROR` on startup and config sync and
  emits `senhub.agent.license.invalid{reason} = 1` on every output. Alert on
  `senhub.agent.license.invalid > 0` to catch any host left on a compact key.
- **Migration — before upgrading a paid host.** Obtain a JWT licence for the
  agent's authentication key, set `agent.license` (inline or
  `${file:/etc/senhub-agent/license}`), and confirm the log shows
  `License validated successfully` with the expected tier. Full steps in the
  0.3.0 note.

No action is required for free-tier installs.

## New

### Connection topology (free tier)

The agent emits its own network position and dependencies as entities, so a
consumer (Toise) can answer "what does this service depend on" and "who connects
to this host" by reading edges rather than reconstructing them.

- **Host interface addresses (#458).** Each non-loopback interface is emitted as
  a `network.interface{host.id, interface.name}` entity and each unicast IP as a
  `network.address{network.address}`, with `network.address --bound_to-->
  network.interface` and `host --has_interface--> network.interface`.
  Loopback / link-local / unspecified / multicast addresses are excluded. This
  is what lets a connection peer resolve back to the host that owns the address.
- **Outbound dependency edges (#457).** From the kernel socket table the agent
  derives durable `service.instance --depends_on--> network.endpoint` edges,
  where the endpoint is keyed on the observable `{server.address, server.port,
  network.transport}`. A connection whose local port is one of the host's own
  listeners is inbound and ignored; the rest are outbound dependencies. Edges
  are debounced over consecutive scrapes and aggregated per distinct peer, so
  the graph carries durable topology, not ephemeral flow — no edge attributes
  (connection counts / bytes stay telemetry). The dependent `service.instance`
  is minted from the local owning process; a connection whose owner cannot be
  named is skipped rather than attributed to a guessed identity.

These ride the existing OTLP log/entity rail (`entities.enabled: true`); the
emission cadence is the `entities.interval` (default 60s).

## Fixed

- **Outbound dependents are anchored to their host.** The minted
  `service.instance` for each outbound dependency now carries a `runs_on` edge
  to its host (the same pattern as `service.listener` and the foundation
  service.instance), so dependents are part of the host's subgraph instead of
  floating attached only by their `depends_on` edge.
- **Entity emission survives a config reload (#495).** A config change that
  recreated the OTLP strategy (for example an endpoint edit) previously started
  the new instance before tearing the old one down. Because the entity sources
  and the single detector are process-global, the overlap left a stale producer
  running alongside the fresh one and silently stopped the entity rail until a
  full restart — while metrics kept flowing, making it invisible. The replaced
  strategy is now shut down before its replacement starts, the clean hand-off a
  manual restart gives. A regression test drives a real strategy recreate and
  asserts entity emission resumes on the new endpoint without a restart.

## Known follow-ups

- **Configurable debounce threshold** (#491): the `depends_on` debounce is a
  package constant (default 3 consecutive scrapes); expose it as an entities
  config knob so operators can trade durability latency against responsiveness.
- **hostdep scan under load** (#492): the dependency source reads the full
  socket table each cycle (it cannot cache like the listener source — the
  debounce needs consecutive scrapes); profile and bound the cost on hosts with
  very large socket tables.
- **Agent self-identity duplication** (#494): the agent's own process appears as
  two `service.instance` nodes — the foundation identity (by agent key) and the
  hostdep identity (`<exe>@<host.id>`). Both are correctly anchored; reconciling
  them to a single canonical node is deferred (it touches the frozen foundation
  identity contract).
- The 0.3.0 follow-ups (#479 Toise type registration, #482 entity invariant
  coverage, #486 paid-licence migration, #489 registration-coverage guard)
  remain tracked.
