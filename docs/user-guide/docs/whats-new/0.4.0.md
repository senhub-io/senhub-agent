# SenHub Agent 0.4.0

:material-calendar: 2026-06-22 · the entity model release

This release rounds out the entity model. The host entity now carries a
full nameplate — OS, CPU and hardware identity plus capacity,
virtualization and chassis attributes (AT10-AT13) — network interfaces
become first-class `network.interface` entities, and Hyper-V guests are
enriched as `compute.vm`. New relations tie the graph together:
listeners `bound_to` an address and `listens_on` an interface, and
local databases and containers `runs_on` their host. Operators can
stamp their own governance attributes on hosts and network devices.
The distributed build matrix also drops to Linux amd64/arm64 and
Windows amd64 — macOS becomes a local-only dev target.

## New

<ul class="rn">
<li><span class="tag t-new">New</span> <span class="tag t-area">entity</span> The host entity now carries a full <strong>nameplate</strong>: OS, CPU and hardware identity, plus <code>host.arch</code>, <code>os.version</code>, <code>os.description</code> (#516) and the AT10-AT12 vocabulary — <code>host.cpu.logical.count</code>/<code>physical.count</code> capacity, <code>host.virtualization</code> and <code>host.chassis.type</code>. (#537, #555)</li>
<li><span class="tag t-new">New</span> <span class="tag t-area">entity</span> Host network interfaces are emitted as first-class <code>network.interface</code> entities with the AT13 descriptive attributes and <code>oper_state</code>; IP-less virtual interfaces (veth and similar) are dropped from the host inventory. (#541)</li>
<li><span class="tag t-new">New</span> <span class="tag t-area">Hyper-V</span> Hyper-V guests are enriched as <code>compute.vm</code> entities — vCPU count, <code>host.virtualization = "hyperv"</code> and the parent <code>host.name</code>; the guest machine-id anchors identity and power state is emitted as the status <code>stateKey</code>. (#456, #561)</li>
<li><span class="tag t-new">New</span> <span class="tag t-area">entity</span> Listeners now attach to the graph through two relations: <code>bound_to</code> a <code>network.address</code> and <code>listens_on</code> a <code>network.interface</code>. (#252, #526)</li>
<li><span class="tag t-new">New</span> <span class="tag t-area">entity</span> Local (loopback) databases and Docker containers anchor to their host through a <code>runs_on</code> relation. (#503, #527)</li>
<li><span class="tag t-new">New</span> <span class="tag t-area">entity</span> Operators can stamp their own <strong>governance attributes</strong> on the host entity and on <code>network.device</code>, driven by a shared attribute vocabulary and discovery rules. (#560)</li>
<li><span class="tag t-new">New</span> <span class="tag t-area">entity</span> Outbound dependency mapping is now opt-in, with a peer CIDR filter (#213) and a configurable <code>depends_on</code> debounce threshold. (#491)</li>
<li><span class="tag t-new">New</span> <span class="tag t-area">SNMP</span> <code>snmp_poll</code> enriches <code>network.device</code> and interface entities with descriptive attributes, and reconciles a device's neighbour-view back to its canonical id. (#565)</li>
</ul>

## Improved

<ul class="rn">
<li><span class="tag t-improved">Improved</span> <span class="tag t-area">databases</span> Database entities carry richer attributes: <code>db.system.version</code> is threaded onto the Redis (#509), MySQL, PostgreSQL and Oracle (#510) entities; the PostgreSQL role attribute is renamed <code>replication.role</code> (#512); and the MySQL/PostgreSQL environment splits into <code>db.deployment.platform</code>. (#519)</li>
<li><span class="tag t-improved">Improved</span> <span class="tag t-area">entity</span> Service entities emit <code>service.version</code> for NATS, Envoy, nginx and Jenkins (#524); Docker containers emit their status as an entity <code>stateKey</code>. (#514)</li>
</ul>

## Fixed

<ul class="rn">
<li><span class="tag t-fixed">Fixed</span> <span class="tag t-area">entity</span> Service and database entities anchor correctly: a <code>monitors</code> edge is emitted for Kubernetes, Modbus and UniFi (#506), and single-key database identity is used for Oracle (#505) and PostgreSQL (#504) so the edge resolves. (#507)</li>
<li><span class="tag t-fixed">Fixed</span> <span class="tag t-area">entity</span> The agent's own dependencies are attached to its foundation node instead of floating unanchored. (#494)</li>
<li><span class="tag t-fixed">Fixed</span> <span class="tag t-area">entity</span> Host-local IPs are no longer emitted as a shared <code>network.address</code>; container-bridge addresses are also skipped by interface name.</li>
<li><span class="tag t-fixed">Fixed</span> <span class="tag t-area">docs</span> License tiers corrected for <code>mysql</code>, <code>postgresql</code> and <code>wifi_signal_strength</code> in the probe catalog.</li>
</ul>

## Changed

<ul class="rn">
<li><span class="tag t-removed">Changed</span> <span class="tag t-area">CI</span> The distributed build matrix is now three platforms — Linux amd64, Linux arm64 and Windows amd64. The macOS test job is dropped and darwin becomes a <strong>local-only</strong> dev/test target, never built or published by CI. (#531)</li>
</ul>

<ul class="rn">

<li><span class="tag t-internal">Internal</span> <span class="tag t-area">tests</span> A load harness measures the socket-scan cost of host dependency detection. (#492)</li>
<li><span class="tag t-internal">Internal</span> <span class="tag t-area">docs</span> The probe catalog is rewritten as an interactive tile catalog with official product logos.</li>
