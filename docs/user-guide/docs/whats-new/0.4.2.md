# SenHub Agent 0.4.2

:material-calendar: 2026-06-30 · entity stabilization & install hardening

A stabilization release for the entity model and the install path. Entity
emission gains liveness, provenance and an anti-orphan guard so consumers stop
flapping on transient gaps, and Windows hosts finally carry a hardware
nameplate. The Pro-tier probe catalogue is corrected so paid probes unlock as
licensed, systemd install and refresh are hardened, and auto-update learns the
beta channel and a portable registry base. The rest is portability and CI
hardening.

<div class="rn-filter"></div>


## New

<ul class="rn">
<li><span class="tag t-new">New</span> <span class="tag t-area">entity</span> Windows hosts now carry a hardware nameplate (vendor, model, serial) read from the SMBIOS firmware table. (#563)</li>
<li><span class="tag t-new">New</span> <span class="tag t-area">entity</span> Entity-events carry a per-method instrumentation scope, recording which probe method produced each entity (provenance). (#253)</li>
<li><span class="tag t-new">New</span> <span class="tag t-area">docs</span> Grafana dashboard pack for the IBM i probe — six dashboards, re-cut from the current metric set. (#108)</li>
</ul>

## Improved

<ul class="rn">
<li><span class="tag t-improved">Improved</span> <span class="tag t-area">entity</span> Entity stabilization batch: entities with no relation are dropped before emission (anti-orphan guard), and the liveness slack was widened (3×→5×) to stop consumers flapping on transient refresh gaps. (#454, #310, #524, #536, #492, #482)</li>
<li><span class="tag t-improved">Improved</span> <span class="tag t-area">auto-update</span> Self-update preflights binary writability and surfaces a clear diagnostic when the running binary cannot be replaced, instead of failing mid-update. (#377)</li>
</ul>

## Fixed

<ul class="rn">
<li><span class="tag t-fixed">Fixed</span> <span class="tag t-area">license</span> The six Pro-tier probes are now registered in the paid catalogue — they were absent, so a valid Pro licence did not unlock them. (#595)</li>
<li><span class="tag t-fixed">Fixed</span> <span class="tag t-area">license</span> Legacy compact (<code>SH-XXXX</code>) licences now produce a clear typed error instead of an opaque failure. (#486)</li>
<li><span class="tag t-fixed">Fixed</span> <span class="tag t-area">auto-update</span> The <code>latest</code> alias honours <code>include_beta</code>, so the beta channel resolves the newest beta release. (#597)</li>
<li><span class="tag t-fixed">Fixed</span> <span class="tag t-area">auto-update</span> The registry base is scaffolded without a trailing <code>/releases</code> and the list status is guarded against malformed responses. (#586, #585)</li>
<li><span class="tag t-fixed">Fixed</span> <span class="tag t-area">install</span> <code>agent install</code> fails when the managed binary cannot be staged, instead of pointing <code>ExecStart</code> at the invocation path. (#576)</li>
<li><span class="tag t-fixed">Fixed</span> <span class="tag t-area">systemd</span> <code>refresh-unit</code> preserves the installed <code>User</code> and creates it before switching. (#575)</li>
<li><span class="tag t-fixed">Fixed</span> <span class="tag t-area">systemd</span> <code>StartLimitIntervalSec</code>/<code>StartLimitBurst</code> moved to the <code>[Unit]</code> section so systemd honours them. (#577)</li>
<li><span class="tag t-fixed">Fixed</span> <span class="tag t-area">network</span> The Windows network probe tolerates <code>PDH_NO_DATA</code>: PDH priming is deferred to the first <code>Collect</code> and the priming sample no longer fails construction on a host with no initial data. (#590)</li>
<li><span class="tag t-fixed">Fixed</span> <span class="tag t-area">outputs</span> The shipped default <code>nagios.yaml</code> is now portable across install paths. (#335)</li>
<li><span class="tag t-fixed">Fixed</span> <span class="tag t-area">docs</span> Corrected a broken GitHub repository URL (<code>sen-hub</code> → <code>senhub-io</code>).</li>
</ul>

<li><span class="tag t-internal">Internal</span> <span class="tag t-area">test</span> Proxmox now covers the <code>up</code> availability metric, matching the Kubernetes probe parity. (#593)</li>
