# Release 0.1.98-beta

**Date:** 2026-05-19
**Type:** Beta Release (CI hardening + privilege gate fix)
**PRs merged to dev:** #129

## Overview

Quality-pass release. No new feature; the change set is entirely focused on widening the CI safety net and removing a usability rough edge in the CLI.

## CI gates added

The `Go-test` workflow gains four new jobs that catch the classes of regression a single-OS `make test` can't see:

| Job | Surface | What it catches |
|---|---|---|
| `test (macos-latest)` | macOS matrix entry | Build-tag drift in platform stubs (e.g. the ibmi `platform_other.go` shipped in 0.1.97-beta), darwin syscall / path mismatches |
| `race` | `go test -race` against every package except the documented flakes in `internal/agent/services/configuration` | New data races introduced by a PR |
| `vuln` | `govulncheck ./...` | Callable CVEs in the resolved dependency graph (direct + transitive). Currently 0 callable findings. |
| `boot-smoke` | new `make boot-smoke` target gated behind a `boot_smoke` build tag | Linker errors, config schema breakage, platform-gated probe drift. Builds the agent binary and asserts `version` + `config check examples/example-config-free-tier.yaml` exit cleanly. |

The race job uses `xargs go test -race` rather than `go test -race $(go list ./...)` because bash unquoted-variable word-splitting was concatenating the multi-line package list into a single malformed import path on the `go test` command line.

The race job EXCLUDES `internal/agent/services/configuration` because that package has 4 pre-existing race flakes (tracked in CLAUDE.md) that pass under plain `make test` but fire under `-race` parallel scheduling. Excluding only that package keeps the race signal tight without papering over real failures elsewhere.

The boot-smoke is on its own job because the inner `go build` it invokes was contending with the outer `go test ./...` build cache under `-race` parallel scheduling, producing false-positive race-detector failures in `cmd/agent`. A separate job keeps the "one CI gate, one signal" property.

## CLI fix

The agent binary refused to run when invoked without root privileges, even for diagnostic subcommands that only display state. After this release:

| Subcommand | Requires root |
|---|---|
| `agent version` | **no** |
| `agent --help`, `agent -h`, `agent help` | **no** |
| `agent config check <path>` | **no** |
| `agent config show <path>` | **no** |
| `agent debug-modules-list` | **no** |
| `agent run`, `install`, `uninstall`, `start`, `stop`, `restart`, `status`, `update`, `db-monitoring`, `ibmi`, no-arg / unknown commands | **yes** (as before) |

The gate is preserved for everything that touches `/var/log/senhub`, `/etc/senhub-agent`, the binary, or the system service. The bug was that the gate was global ‚Äî `version` could not be queried by a non-root user, which broke the new boot-smoke CI job and was a surprise for every contributor running the binary from a clone for the first time.

## Internal / structural

- `cmd/agent/main.go` ‚Äî new `readOnlyCommand(args []string) bool` helper; the privilege gate now runs only when this returns false.
- `cmd/agent/boot_smoke_test.go` ‚Äî new file; gated behind `//go:build boot_smoke`. Builds the binary via `t.Helper`-style helper, exec's `version` + `config check`, asserts exit 0 + no `[ERROR]` lines. Mirrors the `prod_smoke` pattern under `tools/prod_smoke/`.
- `Makefile` ‚Äî new `boot-smoke` target invoking `go test -tags=boot_smoke -v ./cmd/agent/...`.
- `.github/workflows/go-test.yml` ‚Äî single workflow now declares `test` (matrix), `race`, `vuln`, `boot-smoke` jobs. The existing `go-test-windows.yml` is unchanged.

## Operator impact

None for deployed agents. For anyone running `senhub-agent version` or `config check` from a non-root shell, the command now works instead of printing the privilege error.

## Known follow-ups

- 4 race flakes in `internal/agent/services/configuration` remain documented but unfixed. They appear under `make test-race` locally and are filtered out of the CI race job. Fix candidates: shared global state in `LocalConfiguration`, file-watcher goroutine lifetime, the self-signed-cert generator's test.
- The `cmd/agent` test suite occasionally fails under heavy `-race` parallelism (observed once locally before the boot-smoke move to its own job). Not reproduced since. Watch.
